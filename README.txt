# Stampee: Modern Stamp Bidding Platform

"Stampee" is a dynamic web application designed as a school project to serve as an online bidding platform for stamp collectors. Developed with a custom MVC architecture and utilizing MySQL for database management, this project incorporates full CRUD operations to provide a comprehensive user experience for both administrators and users. The project's development was guided by UX/UI principles to cater to a modern, tech-oriented audience.

## Features

- **User and Admin Roles:** Separate functionalities for users and administrators, with admins able to manage user accounts and users able to interact with stamps and bids ('timbre' and 'mise').
- **CRUD Operations:** Comprehensive Create, Read, Update, Delete (CRUD) capabilities for managing stamps (‘timbre’) and bids (‘mise’) for users, and user management for admins.
- **Custom MVC Architecture:** Developed from scratch to specifically address the needs of a stamp bidding website, ensuring flexibility and scalability.
- **Modern UX/UI Design:** The website's design and development process was informed by modern UX/UI best practices, aiming to provide an intuitive and engaging user experience.

## Getting Started

To run Stampee locally, follow these setup instructions:

### Prerequisites

- PHP 7.4 or newer
- MySQL
- Apache or Nginx server

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/picklerules/stampee-final.git
   ```

2. **Navigate to the project directory:**
   ```bash
   cd stampee-final
   ```

3. **Configure the database connection** in the project's configuration files.

4. **Import the provided SQL schema** to your MySQL database to set up the initial structure.

5. **Start your server** and access Stampee through your web browser at the local development URL.

## Project Context

Stampee was conceptualized and developed as a capstone project for a UX/UI class, with the objective to blend technical functionality with cutting-edge design principles. The choice to focus on a stamp bidding platform was predetermined, but the approach to make it appealing to a modern and tech-savvy audience was a critical part of the design and conception process. This involved extensive user research, design iteration, and decision-making to ensure the platform not only met the technical requirements but also delivered a superior user experience.
